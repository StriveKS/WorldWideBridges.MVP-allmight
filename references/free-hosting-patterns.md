# Free Hosting Patterns

## Static-first

Prefer static hosting for MVPs unless the project truly needs a backend.

Options:

- GitHub Pages
- Cloudflare Pages
- Vercel
- Netlify
- Google Sites
- Canva Sites

## Decision guide

### GitHub Pages

Use for docs, simple landing pages and open-source projects.

### Cloudflare Pages

Use for fast static sites and global CDN.

### Vercel

Use for frontend frameworks and Next.js MVPs.

### Netlify

Use for static sites, forms and simple deploy previews.

### Google Sites

Use for non-technical users and internal pages.

### Canva Sites

Use for visual landing pages and fast marketing validation.

## Migration point

Move to a more robust stack when you need:

- authentication;
- database writes;
- server logic;
- payments;
- high customization;
- performance monitoring;
- custom domain and production reliability.
