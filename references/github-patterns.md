# GitHub Patterns

## Default role

Use GitHub for versioning, public distribution, issues, docs and static deploys.

## Patterns

### Public Skill package

```text
SKILL.md
README.md
install.md
templates/
references/
scripts/
examples/
```

### Static landing with GitHub Pages

Use when:

- the project is static;
- no backend is needed;
- the user wants a free public URL;
- versioning matters.

### Issues as backlog

Use GitHub Issues for:

- bugs;
- tasks;
- roadmap;
- contribution review.

### GitHub Actions

Use for:

- automated docs checks;
- static deploy;
- linting;
- simple scheduled jobs.

Avoid when a simple manual process is enough.
