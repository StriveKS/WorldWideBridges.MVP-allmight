# Review Checklist

Use this checklist before publishing a Skill, example, script or tool recommendation.

## Core value

- [ ] Solves a real problem.
- [ ] Useful to more than one person/project.
- [ ] Has a clear target user.
- [ ] Has a clear activation context.

## Free-first quality

- [ ] Includes a 100% free or near-free option.
- [ ] Includes a low-cost robust option when useful.
- [ ] Explains limits of free tools.
- [ ] Does not push paid tools without justification.

## Operational clarity

- [ ] Includes step-by-step workflow.
- [ ] Explains required tools and connections.
- [ ] Says what the LLM can execute or generate.
- [ ] Includes fallback when native connection is unavailable.

## Project memory

- [ ] Updates or references `PROJECT_HISTORY.md`.
- [ ] Includes decisions, problems, solutions and next steps when relevant.
- [ ] Can be transferred between LLMs through `CONTEXTO_LLM.md`.

## Safety and reliability

- [ ] No API keys, passwords or private secrets.
- [ ] No destructive automation without explicit warning.
- [ ] No unsupported claims about current pricing or quotas.
- [ ] Mentions when current information should be verified.

## Publication readiness

- [ ] Has README or explanation.
- [ ] Has examples when needed.
- [ ] Has scripts documented if scripts exist.
- [ ] Has clear next action for the user.
